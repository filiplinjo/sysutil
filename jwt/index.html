Standard JWT Claims Reference

ClaimFull NameDescription
issIssuerIdentifies the principal that issued the JWT.
subSubjectIdentifies the principal that is the subject of the JWT, typically the user ID.
audAudienceIdentifies the recipients that the JWT is intended for. Must be validated by the recipient.
expExpiration TimeUnix timestamp after which the JWT must not be accepted. Used to limit token lifetime.
iatIssued AtUnix timestamp at which the JWT was issued. Used to determine the age of the token.
nbfNot BeforeUnix timestamp before which the JWT must not be accepted. Token is not yet valid.
jtiJWT IDUnique identifier for the token, used to prevent replay attacks.
nameFull NameEnd-user's full name. Defined in OpenID Connect.
emailEmailEnd-user's preferred e-mail address. Defined in OpenID Connect.
rolesRolesApplication-specific claim listing the user's roles or permissions.
