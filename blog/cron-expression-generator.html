What is a Cron Job?

A cron job is a scheduled task in Linux and Unix systems that runs automatically at specified times or intervals. The name comes from Chronos, the Greek god of time. Cron jobs are managed by the cron daemon, a background process that wakes up every minute and checks if any scheduled task needs to run.

Common uses include database backups, log rotation, sending emails, running scripts, clearing cache, and syncing files.

Cron Expression Syntax

A cron expression consists of five fields separated by spaces:

*  *  *  *  *command to execute
│  │  │  │  │
│  │  │  │  └Day of week (0-7, Sunday=0 or 7)
│  │  │  └Month (1-12)
│  │  └Day of month (1-31)
│  └Hour (0-23)
└Minute (0-59)

Special Shorthand Strings

Instead of five fields, cron also accepts these convenient shortcuts:

ShorthandEquivalentDescription
@hourly0 * * * *Once per hour
@daily0 0 * * *Once per day at midnight
@weekly0 0 * * 0Once per week on Sunday
@monthly0 0 1 * *Once per month on the 1st
@yearly0 0 1 1 *Once per year on January 1st
@reboot—Once at system startup

How to Edit Your Crontab

# Open your personal crontab for editing
crontab -e

# List your current cron jobs
crontab -l

# Remove all your cron jobs (careful!)
crontab -r

# Edit crontab for a specific user (as root)
crontab -u username -e

💡 Tip: The first time you run crontab -e it will ask you to choose an editor. Pick nano if you are not familiar with vim.

Crontab File Format

Each line in your crontab is one scheduled job:

# Minute Hour Day Month Weekday Command
# ─────── ──── ─── ───── ─────── ──────────────────────────

# Run backup every day at 2am
0 2 * * * /home/user/backup.sh

# Clear temp files every Sunday at midnight
0 0 * * 0 rm -rf /tmp/myapp/*

# Send report every weekday at 9am
0 9 * * 1-5 /usr/local/bin/send-report.py

# Restart app every 6 hours
0 */6 * * * systemctl restart myapp

System-Wide Cron Jobs

For system-wide cron jobs that run as root, use the /etc/crontab file or drop scripts into these directories:

/etc/cron.hourly/    # runs every hour
/etc/cron.daily/     # runs every day
/etc/cron.weekly/    # runs every week
/etc/cron.monthly/   # runs every month

The system /etc/crontab has an extra field for the user to run as:

# Minute Hour Day Month Weekday User    Command
0 2 * * *  root    /usr/local/bin/backup.sh

Redirecting Cron Output

By default cron emails output to the system user. Redirect it to a log file instead:

# Discard all output
0 2 * * * /path/to/script.sh > /dev/null 2>&1

# Log output to a file
0 2 * * * /path/to/script.sh >> /var/log/myjob.log 2>&1

# Log with timestamp
0 2 * * * echo "$(date): starting" >> /var/log/myjob.log && /path/to/script.sh >> /var/log/myjob.log 2>&1

⚠️ Warning: Always use full paths in cron jobs. The cron environment has a minimal PATH, so commands like python or node may not be found. Use /usr/bin/python3 instead.

Debugging Cron Jobs

# Check if cron daemon is running
systemctl status cron

# View cron logs (Ubuntu/Debian)
grep CRON /var/log/syslog

# View cron logs (CentOS/RHEL)
grep CRON /var/log/cron

# Test your script manually first
bash -x /path/to/script.sh

Environment Variables in Cron

Cron runs with a minimal environment. Set variables at the top of your crontab:

SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
MAILTO=admin@example.com

# Now your jobs
0 2 * * * /usr/local/bin/backup.sh

Frequently Asked Questions

How do I run a cron job every 5 minutes?

Use the step operator: */5 * * * * /path/to/script.sh. The */5 in the minute field means "every 5 minutes".

How do I run a cron job every hour?

Use 0 * * * * /path/to/script.sh — the 0 in the minute field means it runs at the start of every hour.

What is the difference between crontab and cron?

cron is the daemon (background service) that executes scheduled tasks. crontab is both the file that stores the schedule and the command used to edit it.

Can cron run a job every 30 seconds?

Not directly — cron has a minimum resolution of 1 minute. To run every 30 seconds, schedule the job every minute and use sleep: * * * * * /path/script.sh; sleep 30; /path/script.sh.
